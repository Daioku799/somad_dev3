## 目的
 - H2-main_TSV_Optを調査し、プログラム拡張に必要な要件を抽出し、再構成する。

 - H2-mainのmodelA.jlによって与えられているモデルを、gds2やjsonによって簡単にパラメータ設定できるようにする。

## 要件
### モデル
#### 物理構造(研究ドキュメントを引用)
本モデルは、3層のシリコンチップ（Silicon Die）とチップ間を充填するアンダーフィル樹脂（Underfill）による積層構造を基本としている。
最上面には放熱を担うヒートシンク（Heatsink）が配置され、最下面にはプリント配線基板（PCB substrate）が接合されている。
各シリコンチップ層を垂直方向に貫通し、層間の電気的・熱的接続を確保するようにTSV（Through-Silicon Via）が配置される。
各チップ層のTSV間は、はんだバンプ（Solder Bump）によって物理的に接続される構成となっている。

各部品は、複雑な細部を省いても、熱伝導に寄与する主要な断面積と体積を維持していれば、マクロな温度分布に与える影響は無視できるほど小さくなる。
これに基づき本研究でのモデルは、単純な直方体、球体、円柱によるプリミティブ表現で構成され、その形状データは、形状タイプと座標と大きさによる最低限の量のパラメータで表現される。
このプリミティブ表現と外部ファイルによるモデリングの組み合わせによって、TSVの位置や本数、層構造といった設計変数をプログラム外から容易に変更可能である。
これによって、座標や半径といった数値を変更するだけでプログラム上でのモデルを即座に再構成したり、TSV同士の距離制限やチップ範囲などの物理的制約を単純な数式として記述でき、最適化のパラメータとして利用しやすくなる。
さらに、幾何形状を数学的に厳密なプリミティブとして定義することで、複雑な実設計データに起因する境界の不確定性や隙間を排除している。
これにより、格子点がどの部品包含されているかを一意に決定できるため、設計パラメータを大幅に変更する最適化プロセスにおいても、格子生成に失敗することのない安定したマッピングが可能となっている。

#### 格子
本モデルの大きさは1.2mm×1.2mm×0.6mmであり、各方向について240×240×31個の格子に分割する。
xy平面に関しては等間隔に格子が作られるが、Z方向の格子分割については各層の境界付近のみに格子の境界を集中させた不等間隔格子を用いる。
これは、積層構造を有する3D-ICにおいて、Z方向の温度分布が各層の内部（単一媒質）で概ね線形に変化する特性に基づき、線形分布を再現できるような最小格子数と格子幅を設定している。


### パラメータ
 - TSVの位置・直径・本数
     - ハンダバンプは各層のTSVを接合するので、TSVの入力に合わせて生成しなければならない。ハンダバンプの生成ロジックについては要検討。
     - 無効な入力(モデル外に飛び出す,TSV同士が干渉する等)に対するエラー判定
     - 合計本数、総体積等の制約条件の設定(JSONで変更できるようになっていると良い)
 - 各層の厚さ・物性値
 - 熱源配置・シリコンチップ形状

### 実装
#### データフォーマット
 - 最適化プロセスにおける柔軟な形状変更と実設計データとの整合性を両立させるため、JSON形式の設定ファイルとGDSIIフォーマットを併用したハイブリッドなモデリング手法を採用している。
     - 積層構造（各層の厚みや材料物性）およびTSV・はんだバンプの幾何パラメータ（半径、配置座標、個数）は、JSONファイルによってテキスト形式で管理される。
     - シリコンチップ、熱源の形状はgds2によって与えられる。

 - gds2は2Dデータで厚みを持たないため、シリコンチップの形状データを定義するにはz方向格子間隔のデータを参照する必要がある。

 - 熱源はシリコンチップ表面に現れるので、図形の内外判定によってではなく、シリコンチップの上部表面のセルをgds2の形状を参照しながら定義する。

#### プロット
 - H2-mainと同様のプロット
      - 解析結果の温度等高線図
      - TSVが正しく作成できているか確認するためのyz断面図
      - 熱源・チップ・TSVが正しく作成できているか確認するためのxy断面図
      - z方向の温度折れ線グラフ

#### 格子の図形内外判定
 - プリミティブ表現による図形は幾何的に判定できる。H2-main-originalと同様の判定ロジックを用いよ。(優先)
 - gds2によって形状が与えられるシリコンチップは、複雑な形状によって判定に失敗する可能性があるので別の判定ロジックを用いる。ray casting法を考えていたが、三角形パッチに変換して判定する方法も模索する。
 - 熱源はシリコンチップ上部表面のみに与えるので内外判定する必要はない

## 絶対条件（プロジェクト再建の理念）
 - **IDマップ生成以降のプロセスの完全一致**: 本プロジェクトの拡張はモデル生成（IDマップ構築）までとし、それ以降の熱解析プロセス（ソルバー、収束判定等）は `H2-main-original` と完全に一致させなければならない。
 - **オリジナルコードの最大限の流用**: プリミティブの内外判定ロジックや座標計算は、オリジナルのコードをそのまま流用または等価な形で継承する。
 - **H2-main-copy内での開発**: 実装は `H2-main-copy` ディレクトリ内の既存ファイルを上書き・追加・変更することで行い、ソルバー呼び出し側の `main.jl` 等の構成を変更しない。
 - **数値的一致の保証**: 標準的な入力パラメータを与えた際、出力される ID マップ、物性値マップ、座標配列はオリジナルの出力とビットレベルで一致（または実数誤差の範囲内で極めて近似）していなければならない。不収束を避けるため、この整合性を最優先する。

## 動作試験
以下の順に動作試験を行う。各試験において、パラメータ（TSV 本数、配置、各層の厚み等）は `H2-main-original` または `H2-main_TSV_Opt` で使用されているものと完全に一致させること。

1. H2-main-originalとH2-main_TSV_Optは同じモデルの入力に対して同じ温度場を返した。今回の拡張でも同様のモデルの入力を行う。
2. 同じ本数、同じ直径で、配置のみを変えたもの
3. 小さいものを本数を増やしたもの
4. 大きいものを一本# somad_dev3
