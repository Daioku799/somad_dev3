# Requirements: validation-plot

## 1. 設定駆動型のプロット生成
- 1.1 **[JSON指定のスライス]** プロットモジュールは、`config.json` で指定された断面座標（X, Y, Z）および注目レイヤーに基づいて、自動的にプロットを生成しなければならない。
- 1.2 **[物理スケーリング]** 全てのプロットにおいて、座標軸は実際の物理寸法（mm）を反映しなければならない。

## 2. 可視化対象の拡大
- 2.1 **[材料分布プロット]** 材料 ID ごとに識別しやすい色を割り当てた 2D 断面図を生成しなければならない。
- 2.2 **[温度分布プロット]** 解析ソルバーから出力された温度データに対し、等高線図（Contour Plot）またはヒートマップを生成しなければならない。

## 3. GDS 整合性検証 (GDS Overlay)
- 3.1 **[ポリゴン重ね書き]** 格子マッピングの正確性を検証するため、ID マップの断面図の上に、ソースとなった **GDSII ポリゴンの境界線（生データ）を重ねて描画**する機能を提供しなければならない。
- 3.2 **[不整合の検出]** ユーザーが格子点（ピクセル）と GDS 境界線の視覚的なズレを確認できるよう、対比可能な描画形式（透過処理等）をサポートしなければならない。

## 4. データ管理
- 4.1 **[画像エクスポート]** 生成された全ての検証画像は、指定されたディレクトリ（例：`plots/`）に、パラメータ情報を付随させたファイル名で保存されなければならない。

## Scope Boundaries
- **In**: 設定ファイルに基づく断面抽出、ID/温度プロット、GDS境界線のオーバーレイ描画。
- **Out**: リアルタイム 3D プレビュー、対話型 GUI による操作。
