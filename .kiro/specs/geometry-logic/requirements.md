# 要件定義: geometry-logic

## 1. 幾何判定カーネルの提供
- 1.1 **[オリジナルロジックの継承]** ジオメトリエンジンは、`H2-main-original` で採用されている以下の判定ロジックを忠実に継承し、独立した関数として提供しなければならない。
  - `is_included_rect`: 直方体の体積占有判定（幾何学的厳密解）。
  - `is_included_cyl`: 円柱のサンプリング判定。
  - `is_included_sph`: 球体のサンプリング判定。
- 1.2 **[GDS連携]** 複雑な2D形状については、`GdsMapping` モジュールを呼び出し、Z軸方向の範囲チェックと組み合わせた3次元包含判定を提供しなければならない。

## 2. 判定の精度とパフォーマンス
- 2.1 **[サンプリング制御]** 円柱および球体の判定において、オリジナルと同等の精度（samples=50 等）を維持しつつ、必要に応じて計算負荷を調整可能なパラメータ設定をサポートしなければならない。
- 2.2 **[BBox高速化]** 各判定関数は、詳細な幾何計算（サンプリング等）の前に、境界ボックス（BBox）による早期棄却を実行しなければならない。

## 3. 優先順位の外部委守（実行順序依存）
- 3.1 **[シンプル判定]** ジオメトリエンジン自体は点ごとの優先順位テーブルを持たず、純粋な「指定された形状に含まれるか」の真偽値（または占有率）のみを返さなければならない。
- 3.2 **[上書き制御の支援]** 呼び出し側（model-builder）が「空き領域への適用」を容易に行えるよう、一貫した真偽値ベースのインターフェースを維持しなければならない。

## Scope Boundaries
- **In**: オリジナル準拠の幾何判定数学ロジック、BBoxフィルタリング、GDS判定の統合。
- **Out**: 点ごとの優先順位解決（`model-builder`のループ順序が担当）、3D配列の確保。
